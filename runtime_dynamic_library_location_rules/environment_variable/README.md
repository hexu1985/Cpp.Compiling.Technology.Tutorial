### LD_LIBRARY_PATH 环境变量

在库搜索路径概念发展的早期阶段，开发人员迫切需要一种临时、简便且有效的机制来实验和测试他们的设计。
为满足这一需求，系统提供了通过特定环境变量（LD_LIBRARY_PATH）来实现的机制。

当rpath（DT_RPATH）值未设置时，通过此方式提供的路径将作为最高优先级的搜索路径信息。

注意：在此优先级体系中，二进制文件中嵌入的值与环境变量之间存在着不对等的竞争关系。
如果维持现状，二进制文件中rpath的存在将使得第三方软件产品的故障排查变得不可能。
幸运的是，新的优先级方案通过认识到rpath过于专断，并提供了临时覆盖其设置的方案，从而解决了这个问题。
rpath的年轻兄弟runpath被赋予了遏制专横rpath的能力，在这种情况下，LD_LIBRARY_PATH才有机会临时获得最高优先级的待遇。

设置LD_LIBRARY_PATH的语法与设置任何路径变量的语法相同。例如，可以在特定shell实例中通过输入以下命令来实现：

```bash
$ export LD_LIBRARY_PATH=/home/milan/projects:$LD_LIBRARY_PATH
```

需要再次强调，此机制应仅用于实验目的。软件产品的生产版本完全不应依赖此机制。
